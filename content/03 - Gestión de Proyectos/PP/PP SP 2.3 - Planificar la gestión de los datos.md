## Información del Departamento (Notion)

### Objetivo de la Práctica
El objetivo de esta práctica es identificar, recopilar, almacenar, y proteger los datos que se generan durante el proyecto.

### Resumen de la Subpráctica
Los datos son todas las formas de documentación requeridas para un proyecto. Es indispensable una razón clara para cada recolección de documentos por su alto costo. Debe de haber requisitos de datos para los elementos a crear como para su contenido, forma y forma de recolectarlos. Para ello se recomienda: Establecer requisitos y procedimientos para la seguridad y privacidad de los datos. Mecanismos para almacenar y acceder a los datos. Determinar datos a identificar, recoger y distribuir. Requisitos para dar acceso a los datos a los stakeholders. Determinar que datos requieren control de versiones y mecanismos para controlar los datos.

### Artefactos que Cumplen con la Práctica
- Plan de Gestión de datos. Requisitos de Privacidad. Requisitos de Seguridad. Proceso para recuperación
- reproducción y distribución de datos.
