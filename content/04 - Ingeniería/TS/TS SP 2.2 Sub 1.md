**Área de Proceso:** [[TS]] · **Meta:** [[TS SG 2 - Desarrollar el diseño|SG 2]] · **Práctica:** [[TS SP 2.2 - Establecer un paquete de datos técnicos|SP 2.2]]

## Descripción

Determinar el número de niveles de diseño y el nivel apropiado de documentación para cada nivel de diseño. Determinar el número de niveles de componentes de producto (p. ej., subsistema, elemento de configuración del hardware, tarjeta de circuitos, elemento de configuración de software de ordenador [CSCI], componente de producto software de ordenador, unidad software de ordenador) que requieren documentación y trazabilidad de requisitos, es importante para gestionar los costes de la documentación y dar soporte a los planes de integración y verificación.
