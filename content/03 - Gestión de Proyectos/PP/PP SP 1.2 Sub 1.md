# PP SP 1.2 — Subpráctica 1

**Área de Proceso:** [[PP]] · **Meta:** [[PP SG 1 - Establecer las estimaciones|SG 1]] · **Práctica:** [[PP SP 1.2 - Establecer las estimaciones de los atributos de los productos de trabajo y de las tareas|SP 1.2]]

---

## Descripción

Determinar la aproximación técnica para el proyecto. La aproximación técnica define una estrategia de alto nivel para el desarrollo del producto. Incluye las decisiones sobre las características de la arquitectura, tales como estructura distribuida o cliente/servidor; estado del arte o tecnologías existentes para aplicarse, tales como robótica, materiales compuestos o inteligencia artificial; y atributos de funcionalidad y de calidad esperados en los productos finales, tales como seguridad, protección y ergonomía.

## Contexto del Departamento

> **Objetivo (Departamento):** Identificar atributos cuantitativos para las estimaciones de los requisitos del proyecto para determinar el esfuerzo, el coste y el calendario del proyecto
>
> **Artefactos:** RTM - Requirement Traceability Matrix (define todas las historias de usuario y desglosa su complejidad), SRS - Software Requirement Specification (establece el número de requisitos funcionales), SAD - Software Architecture Document (establece la arquitectura y la dificultad de implementación), WBS - Work Breakdown Structure (establece toda la descomposición de los entregables para completar un proyecto), Backlog (seguimiento de las historias de usuario)
